31 ### solvers
32 
33 Ultimately our goal is to implement the most recent convex solvers so we can mix and match control problem formulations with different solver algorithms. Right now we have only implemented a slightly modified version of QuadProg.
34 
35 #### quadprog
36 
37 This library is a QP (Quadratic Program) based on the QuadProg++ project (http://quadprog.sourceforge.net/) which has been slightly modified. In this version, vector and matrix classes are replaced by Eigen classes, in order to use the same definitions as the ocra libraries.

41 #### qpoases
42 
43 Exploits the open-source C++ library qpOASES, which is an implementation of the recently proposed online active set strategy, which was inspired by important observations from the field of parametric quadratic programming (QP). It has several theoretical features that make it particularly suited for model predictive control (MPC) applications but also as a QP solver. [(ref)](https://projects.coin-or.org/qpOASES)

72 ## Installation
73 
74 **WARNING**
75 *This is an experimental set of libs and there are no guarantees that they will not do damage to your computer. We take no responsibility for what happens if you install them. That being said, if you follow these instructions you should be fine.*
76 
77 Okay that's out of the way... phew!
78 
79 **Install to /usr/local**
80 ```
81 git clone https://github.com/ocra-recipes/ocra-recipes.git
82 cd ocra-recipes
83 mkdir build
84 cd build
85 cmake ..
86 sudo make install
87 ```
88 
89 **Install to custom location (example: /home/user/Install)**
90 ```
91 git clone https://github.com/ocra-recipes/ocra-recipes.git
92 cd ocra-recipes
93 mkdir build
94 cd build
95 cmake -DCMAKE_INSTALL_PREFIX=home/user/Install ..
96 make install
97 ```
98 *Note:* If you do the install this way then you must update your environment variables in your `.bashrc` file.
99 
100 **In `.bashrc`**
101 ```shell
102 # ocra-recipes install
103 export OCRA_INSTALL=/home/user/Install
104 export LD_LIBRARY_PATH=${LD_LIBRARY_PATH}:${OCRA_INSTALL}/lib
105 export LIBRARY_PATH=${LIBRARY_PATH}:${OCRA_INSTALL}/lib
106 export PATH=${PATH}:${OCRA_INSTALL}/bin
107 ```

125 ### Notes about OS X
126 :warning: As a recent feature we added rpath support, therefore, by default the flag `OCRA_ICUB_ENABLE_RPATH` is `OFF`. Change it by configuring `ocra-wbi-plugins` as: `cmake -DOCRA_ICUB_ENABLE_PATH=ON ./`. Once we're sure this ''always'' works it will be `ON` by default. Therefore, if you encounter error messages such as:
127 
128 ```bash
129 dyld library not loaded @rpath/libYarpMath.dylib
130 ```
131 
132 Most likely this variable is still `OFF`.
133 
134 :warning: When using XCode to debug your code, make sure you change your LLVM C++ Language settings manually to C++11, by heading to the `Build Settings` of your project, searching for `C++ Language Dialect` and changing it to `C++11 [-std=c++11]`

141 ### Generating the documentation
142 
143 To build the documentation for `orca-recipes` you will need [`doxygen`](http://www.stack.nl/~dimitri/doxygen/index.html). To install `doxygen` on linux run `sudo apt-get install doxygen`.
144 
145 In your `build/` directory (if you have already run `cmake ..` simply run:
146 ```
147 make doc
148 ```
149 **
150 If you run this right now you must ignore the latex errors due to the missing `.sty` files by just holding down `enter` until they are past.
151 **
152 
153 HTML and LaTeX files will be generated in the `build/docs/` directory. Open the file, `index.html`.
